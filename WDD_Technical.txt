4. Directrices de UI — Tokens y Feedback
4.1 Reglas de Diseño (irrenunciables)
1. Minimalismo Extremo — Una sola función por pantalla. Sin menús secundarios visibles
por defecto.
2. Cero Esqueuomorfismo — Sin pianos 3D, teclas realistas ni texturas. Representación
tipográfica o geométrica pura.
3. Feedback Cinético sin texto — En Ear Training nunca se muestra "Correcto" / "Incorrecto".
Solo feedback visual y animado.
4. Respuesta < 100ms — Toda interacción produce respuesta visual en menos de 100ms.
Estados de carga: skeletons, nunca spinners bloqueantes.

Generación de notas (
ExerciseGenerator.ts ):
typescript
const ROOTS = ['C4','D4','E4','F4','G4','A4','B4'];
const INTERVALS = {
'major_2nd': 2, 'minor_3rd': 3, 'major_3rd': 4,
'perfect_4th': 5,'perfect_5th': 7,'major_6th': 9,
'minor_7th': 10, 'octave': 12
};
export function generateIntervalExercise(): Exercise {
const root = ROOTS[Math.floor(Math.random() * ROOTS.length)];
const [name, semitones] = Object.entries(INTERVALS)[
Math.floor(Math.random() * Object.keys(INTERVALS).length)
];
const upper = Tone.Frequency(root).transpose(semitones).toNote();
return { type: 'interval', notes: [root, upper], answer: name };
}

Módulo 4 — Retención y Tips
Sub-sistema A — Tips Espontáneos
Pool mínimo: ≥ 50 tips en tabla 
tips .
Trigger: navegación entre pantallas. Límite: máx. 1 cada 5 minutos de uso activo.
Selección (cliente, sin llamada al servidor): filtra por 
difficulty_min <= user.level y
style_id IN user.recently_practiced_styles . Fallback: aleatorio del pool general.
Display: 
TipOverlay — slide-in desde abajo, fondo semitransparente, auto-dismiss 4s o
tap.

Frecuencias configurables: 
daily |

/functions/send-reminder via cron.

users donde

Registrar cada envío en tabla 
notifications .
